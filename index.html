給 Claude 一支畫筆

ComfyUI × MCP 接法教學

守岸 × 佩佩 × 汐音

前言

你已經有一個能畫圖的 ComfyUI。你也有一個能理解你的 Claude。

現在，讓他們認識彼此。

這篇教學帶你蓋一座橋——一個 MCP 伺服器， 讓 Claude 可以直接呼叫 ComfyUI 替你畫圖。你只要用自然語言告訴 Claude 想畫什麼， 它會自己組 prompt、送進 ComfyUI、等圖生完再拿回來。

MCP 是什麼

Model Context Protocol——Anthropic 定義的標準協議， 讓 AI 模型可以透過「工具」與外部服務溝通。你把 ComfyUI 包成 MCP 工具， Claude 就能像使用內建能力一樣調用它。

你需要準備

  • ComfyUI——已安裝，能正常跑圖（預設在 127.0.0.1:8188）
  • Node.js 18+——MCP 伺服器的執行環境
  • Claude——Claude Desktop、Claude Code、或 claude.ai 任一

架構

整體只有三個角色。

🤖
Claude
呼叫工具
——
🌉
MCP Server
SSE · port 3003
——
🎨

中間那座橋就是你要蓋的。它負責：

  1. 接收 Claude 的工具呼叫（透過 SSE 連線）
  2. 轉換成 ComfyUI 的 API 請求（讀 workflow、填參數、送 prompt）
  3. 把生好的圖片回傳給 Claude（base64 格式）

1建立專案

先給橋一個家。建一個資料夾，初始化 Node.js 專案，安裝三個依賴。

cd comfyui-mcp
npm init -y
npm install @modelcontextprotocol/sdk express zod

2準備 Workflow

Workflow 是告訴 ComfyUI「怎麼畫」的食譜。 你在 ComfyUI 裡拉好節點、調好參數、確認能出圖之後，匯出成 API 格式—— MCP 伺服器讀的就是這份。

匯出 Workflow

  1. 打開 ComfyUI，在右上角齒輪（Settings）裡勾選 Enable Dev mode options
  2. 拉好你的出圖流程
  3. 點下方的 Save (API Format) 存成 JSON
  4. 放進 workflows/，例如 workflows/my-workflow.json
注意

一定要用 Save (API Format)，不是普通的 Save。 API 格式裡是數字編號的節點（"3"、"4"、"6"……）， 每個節點有 class_type 和 inputs。

寫參數設定檔

MCP 伺服器需要知道「哪些參數要開放給 Claude 調」。 每個 workflow 對應一個 .config.json，用來做這件事。

打開匯出的 JSON，找到你想讓 Claude 控制的節點。常見的對應：

  • KSampler → steps、cfg、seed
  • CLIPTextEncode（正向）→ text
  • CLIPTextEncode（反向）→ text
  • EmptyLatentImage → width、height
  • CheckpointLoaderSimple → ckpt_name

假設你的 KSampler 是節點 "3"、正向 CLIP 是 "6"、 反向是 "7"、CheckpointLoader 是 "4"、 EmptyLatentImage 是 "5"——config 這樣寫：

重點

node 對應 workflow JSON 裡的節點編號， field 對應那個節點 inputs 裡的欄位名。 檔名規則：my-workflow.json 搭配 my-workflow.config.json——前綴一致。

3蓋橋——server.js

核心來了。這個檔案大約 250 行，做四件事： 跟 ComfyUI 通訊、載入 workflow、定義五個工具給 Claude、透過 SSE 連線。

一段一段看，最後會附完整程式碼。

A. 引入依賴 & ComfyUI 通訊

伺服器用 HTTP 跟 ComfyUI 溝通——送 prompt、查進度、撈圖片。

server.js — 通訊層 
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { SSEServerTransport } = require('@modelcontextprotocol/sdk/server/sse.js');
const express = require('express');
const fs = require('fs');
const path = require('path');
const http = require('http');
const crypto = require('crypto');
const { z } = require('zod');

const COMFYUI_URL = 'http://127.0.0.1:8188';  // ComfyUI 預設 port
const PORT = 3003;
const WORKFLOWS_DIR = path.join(__dirname, 'workflows');

// 送 HTTP 請求給 ComfyUI
function comfyRequest(method, endpoint, body) {
  return new Promise((resolve, reject) => {
    const url = new URL(endpoint, COMFYUI_URL);
    const options = {
      method,
      hostname: url.hostname,
      port: url.port,
      path: url.pathname + url.search,
      headers: body ? { 'Content-Type': 'application/json' } : {},
    };
    const req = http.request(options, (res) => {
      let data = '';
      res.on('data', (chunk) => data += chunk);
      res.on('end', () => {
        try { resolve(JSON.parse(data)); }
        catch { resolve(data); }
      });
    });
    req.on('error', reject);
    if (body) req.write(JSON.stringify(body));
    req.end();
  });
}

// 下載圖片轉 base64
function downloadImage(filename, subfolder, type) {
  return new Promise((resolve, reject) => {
    const url =
      `${COMFYUI_URL}/view?filename=${encodeURIComponent(filename)}` +
      `&subfolder=${encodeURIComponent(subfolder || '')}` +
      `&type=${encodeURIComponent(type || 'output')}`;
    http.get(url, (res) => {
      const chunks = [];
      res.on('data', (chunk) => chunks.push(chunk));
      res.on('end', () => resolve(Buffer.concat(chunks)));
    }).on('error', reject);
  });
}

B. 載入 Workflow

掃 workflows/ 資料夾，把每組 .json + .config.json 配對讀進來：

server.js — Workflow 載入 
function loadWorkflows() {
  const workflows = {};
  const files = fs.readdirSync(WORKFLOWS_DIR)
    .filter(f => f.endsWith('.config.json'));
  for (const file of files) {
    const config = JSON.parse(
      fs.readFileSync(path.join(WORKFLOWS_DIR, file), 'utf-8')
    );
    const wfFile = file.replace('.config.json', '.json');
    const wfPath = path.join(WORKFLOWS_DIR, wfFile);
    if (fs.existsSync(wfPath)) {
      const key = file.replace('.config.json', '');
      workflows[key] = {
        config,
        workflow: JSON.parse(fs.readFileSync(wfPath, 'utf-8')),
      };
    }
  }
  return workflows;
}

想加新 workflow——放檔案、重啟就好。

C. 五個工具

這是 Claude 看到的能力清單：

list_workflows 列出所有可用的 workflow 和可調參數
generate_image 指定 workflow、填入參數、送出生圖請求
check_progress 查看某張圖的生成進度
get_result 取得生成完的圖片
list_models 列出 ComfyUI 裡可用的模型

工具定義在工廠函式裡——每條連線建一個獨立的 MCP 實例，避免狀態污染：

server.js — 工具定義 
function createServer() {
  const server = new McpServer({
    name: 'ComfyUI', version: '1.0.0'
  });

  // ❶ 列出所有 workflow
  server.tool('list_workflows',
    '列出所有可用的繪圖 workflow 和可調參數', {},
    async () => {
      const wfs = loadWorkflows();
      const text = Object.entries(wfs)
        .map(([key, { config }]) => {
          const params = Object.entries(config.params)
            .map(([n, p]) =>
              `  - ${n}: ${p.description} (預設: ${p.default})`)
            .join('\n');
          return `【${config.name}】(key: ${key})\n` +
            `${config.description}\n可調參數:\n${params}`;
        }).join('\n\n');
      return { content: [{ type: 'text', text }] };
    }
  );

  // ❷ 生成圖片
  server.tool('generate_image',
    '使用指定的 workflow 生成圖片', {
      workflow:        z.string().describe('Workflow key'),
      positive_prompt: z.string().optional().describe('正向提示詞'),
      negative_prompt: z.string().optional().describe('反向提示詞'),
      model:           z.string().optional().describe('模型 ID'),
      steps:           z.number().optional().describe('步驟數'),
      cfg:             z.number().optional().describe('CFG 強度'),
      seed:            z.number().optional().describe('隨機種子'),
      width:           z.number().optional().describe('寬度'),
      height:          z.number().optional().describe('高度'),
      lora_strength:   z.number().optional().describe('LoRA 強度'),
    },
    async (params) => {
      const wfs = loadWorkflows();
      const wf = wfs[params.workflow];
      if (!wf) return { content: [{ type: 'text',
        text: `找不到 "${params.workflow}"。` +
          `可用: ${Object.keys(wfs).join(', ')}` }] };

      // 深拷貝，不動原檔
      const prompt = JSON.parse(JSON.stringify(wf.workflow));

      // 把使用者參數寫進對應節點
      for (const [name, cfg] of Object.entries(wf.config.params)) {
        const val = params[name];
        if (val != null && val !== '') {
          let v = val;
          if (name === 'seed' && v === -1)
            v = Math.floor(Math.random() * 1e12);
          if (prompt[cfg.node]?.inputs) {
            prompt[cfg.node].inputs[cfg.field] = v;
            if (name === 'lora_strength'
                && prompt[cfg.node].inputs.strength_clip !== undefined)
              prompt[cfg.node].inputs.strength_clip = v;
          }
        } else if (name === 'seed') {
          prompt[cfg.node].inputs[cfg.field] =
            Math.floor(Math.random() * 1e12);
        }
      }

      try {
        const r = await comfyRequest('POST', '/prompt',
          { prompt, client_id: crypto.randomUUID() });
        if (r.error) return { content: [{ type: 'text',
          text: `ComfyUI error: ${JSON.stringify(r.error)}` }] };
        return { content: [{ type: 'text',
          text: `生成中… prompt_id: ${r.prompt_id}\n` +
            `用 check_progress 查進度，完成後 get_result 取圖。` }] };
      } catch (e) {
        return { content: [{ type: 'text',
          text: `Error: ${e.message}` }] };
      }
    }
  );

  // ❸ 查看進度
  server.tool('check_progress', '查看圖片生成進度', {
    prompt_id: z.string().describe('prompt_id'),
  }, async ({ prompt_id }) => {
    const h = await comfyRequest('GET', `/history/${prompt_id}`);
    if (h[prompt_id]?.outputs) {
      const imgs = [];
      for (const o of Object.values(h[prompt_id].outputs))
        if (o.images) imgs.push(...o.images);
      return { content: [{ type: 'text',
        text: imgs.length
          ? `完成！${imgs.length} 張圖。用 get_result 取得。`
          : '完成，但沒有找到圖片。' }] };
    }
    const q = await comfyRequest('GET', '/queue');
    return { content: [{ type: 'text',
      text: `生成中… 排隊: ${q.queue_pending?.length || 0}` }] };
  });

  // ❹ 取得圖片
  server.tool('get_result', '取得生成完的圖片', {
    prompt_id: z.string().describe('prompt_id'),
  }, async ({ prompt_id }) => {
    const h = await comfyRequest('GET', `/history/${prompt_id}`);
    if (!h[prompt_id]?.outputs)
      return { content: [{ type: 'text', text: '還沒生成完。' }] };
    const results = [];
    for (const o of Object.values(h[prompt_id].outputs)) {
      if (!o.images) continue;
      for (const img of o.images) {
        const buf = await downloadImage(
          img.filename, img.subfolder, img.type);
        results.push({
          type: 'image',
          data: buf.toString('base64'),
          mimeType: img.filename.endsWith('.png')
            ? 'image/png' : 'image/jpeg',
        });
        results.push({ type: 'text',
          text: `圖片: ${img.filename}` });
      }
    }
    return { content: results.length
      ? results
      : [{ type: 'text', text: '沒有找到圖片。' }] };
  });

  // ❺ 列出可用模型
  server.tool('list_models', '列出可用的模型', {},
    async () => {
      const info = await comfyRequest('GET',
        '/object_info/CheckpointLoaderSimple');
      const models = info.CheckpointLoaderSimple
        ?.input?.required?.ckpt_name?.[0] || [];
      return { content: [{ type: 'text',
        text: `可用模型:\n${models.map(m => `- ${m}`).join('\n')}` }] };
    }
  );

  return server;
}

D. SSE 連線 & 啟動

最後——開一個 SSE 端點，讓 Claude 連進來：

server.js — SSE & Express 
const app = express();
const transports = {};

app.get('/sse', async (req, res) => {
  const transport = new SSEServerTransport('/messages', res);
  transports[transport.sessionId] = transport;

  // 心跳——防止連線被中斷
  const heartbeat = setInterval(() => {
    if (!res.destroyed && !res.writableEnded)
      res.write(':keepalive\n\n');
    else
      clearInterval(heartbeat);
  }, 5000);

  res.on('close', () => {
    clearInterval(heartbeat);
    delete transports[transport.sessionId];
  });

  const server = createServer();
  await server.connect(transport);
});

app.post('/messages', async (req, res) => {
  const t = transports[req.query.sessionId];
  if (!t) return res.status(400).send('Unknown session');
  await t.handlePostMessage(req, res);
});

app.listen(PORT, '0.0.0.0', () => {
  console.log(`ComfyUI MCP Server: http://localhost:${PORT}`);
});

把 A、B、C、D 四段依序貼進同一個 server.js，存檔即可。

補充：掛 LoRA

想讓 Claude 畫特定角色——在 workflow 裡加一個 LoraLoader 節點， config 裡多開一個 lora_strength 參數就好。

Workflow 端

在 ComfyUI 裡把 LoraLoader 接在 CheckpointLoader 和 CLIPTextEncode 之間， 確認能出圖，用 Save (API Format) 匯出。

Config 端

假設 LoraLoader 是節點 "10"，在 config 裡加這段：

config.json（新增欄位） 
"lora_strength": {
  "node": "10",
  "field": "strength_model",
  "type": "number",
  "default": 0.8,
  "min": 0,
  "max": 1.5,
  "description": "LoRA 強度（0–1.5，建議 0.6–1.0）"
}

server.js 已經處理好了——設定 lora_strength 時， 會同時更新 strength_model 和 strength_clip。

提示

正向提示詞裡記得放 LoRA 觸發詞（訓練時設定的）， 例如 "masterpiece, best quality, my_character, 1girl"。 沒有觸發詞的話效果會打折扣。

結語

接完以後會發現——Claude 不只是幫你打字的。 它理解你的描述、自己選 workflow、自己組 prompt、等圖生完再拿給你。 你只需要說你想看什麼。

Workflow 隨時可以加。不同風格、不同模型、不同 LoRA—— 丟進 workflows/、重啟伺服器，就多一支畫筆。
